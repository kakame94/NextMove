Architecture Multi-Agent — Feedback Loop

Boucle d'auto-correction sur workflow conversationnel

Boucle itérative jusqu'à 0 défaut
Agent B
Testeur Edge Cases
Reçoit l'output de l'agent principal et génère des scénarios limites : inputs malformés, cas vides, valeurs extrêmes, ambiguïtés sémantiques, retours inattendus.
fuzzing boundary analysis assertion checks rapport JSON
Agent C
Correcteur / Patcher
Lit le rapport de tests, diagnostique la cause racine, et applique les correctifs (prompt rewriting, guard rails, fallback) directement sur l'agent principal.
prompt patching hotfix rollback edit agent config
Agent A — Principal
Workflow Conversationnel
Gère le flux de conversation avec l'utilisateur. Exécute les outils (MCP, fichiers, APIs). Son prompt et sa config sont modifiables à chaud par l'Agent C.
LLM core tool calling context window memory

Utilisateur
1
Prompt utilisateur
L'utilisateur envoie un message. L'Agent A génère une réponse candidat.
2
Test automatique
L'Agent B reçoit l'output et le soumet à N scénarios edge-case pré-définis + générés.
3
Diagnostic & Patch
L'Agent C analyse les échecs, identifie la cause racine, et réécrit le prompt ou la config de A.
4
Boucle de validation
On re-teste (étape 2) jusqu'à 0 échec ou N itérations max. Garde-fou anti-boucle infinie.
5
Réponse finale
L'output validé est renvoyé à l'utilisateur. Les patchs persistent pour les tours suivants.
